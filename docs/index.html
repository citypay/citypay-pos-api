CityPay POS API API Reference

CityPay Point of Sale API for payment with card present devices including EMV readers and contactless POS readers.

The API makes it simple to add EMV and contactless card acceptance to iOS, Android, Tablet and desktop applicaitons using a HTTPS protocol. It segregates the complexity of payment processing from the sales environment and eliminates any necessity for the target system to handle card data.

API Endpoint
https://pos.citypay.com/citypay-pos-api/1.0.0
Contact: dev@citypay.com
Schemes: https
Version: 1.0.0

Payment Module

The payment module conducts payment functionality between the host and the target device such as sale, refund and reversal processing.

Payment Module

Sale Transaction

POST /sale

Initiates a new sale to a device. The action will contact the device and request a transaction start including the amount and a unique reference to track the transaction through it's lifecycle.

SaleRequest
undefined
Request Content-Types: application/json
Request Example
{
  "device": "TID001",
  "amount": 4798,
  "identifier": "Tx12345"
}
200 OK
SaleResponse

Starts a sale request to the device. The device should be polled using a GET /transaction call

400 Bad Request

bad input parameter

403 Forbidden
Result

Unauthorized to access the device

Response Content-Types: application/json
Response Example (200 OK) 
{
  "success": "boolean",
  "identifier": "Tx12345",
  "description": "string"
}
Response Example (403 Forbidden) 
{
  "message": "string",
  "success": "boolean"
}

Result: object

message: string

A message related to the request. If the action was successful the message will contain information that may be disaplyed back to the end user or if the action success is false, the message will contain an error message explaining the error. The message will not convey sensitive information.

success: boolean

A boolean result of the action requested

Example
{
  "message": "string",
  "success": "boolean"
}

SaleRequest: object

A request for a sale

device: string

The name of the target device used by the API. The device will be setup prior to the sale but allows targeting of multiple devices.

amount: integer

The amount to process in the lowest denomination and in the base currency of the device. 4798 for instance would process the transaction for £47.98

identifier: string

An identifier or reference for the sale as taken by the point of sale system. This should be used to cross reference transactions after it has processed. The value should be unique and will be used to check on the processing of the transaction.

Example
{
  "device": "TID001",
  "amount": 4798,
  "identifier": "Tx12345"
}
